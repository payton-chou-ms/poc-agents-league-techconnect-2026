```
#### 問題描述
供應商系統資料拋轉延遲，導致網站顯示「無庫存」的假性缺貨問題。

#### 常見原因
1. **API 連線逾時** - 供應商 API 回應時間過長
2. **排程任務失敗** - 同步排程被中斷或卡住
3. **資料格式變更** - 供應商端資料格式異動

#### 解決步驟
1. 檢查同步排程狀態 → Azure Functions
2. 查看錯誤日誌 → Log Analytics
3. 手動觸發同步 → Admin Portal
4. 驗證資料一致性

#### 過往案例
| 日期 | 問題 | 解決方式 | 處理時間 |
|------|------|----------|----------|
| 2025-08-20 | 台灣供應商 API 逾時 | 調整 timeout 設定 | 2 小時 |
| 2025-06-15 | 日本供應商格式變更 | 更新 parser | 4 小時 |

### 💡 結論
**供應商同步延遲是常見問題**，建議檢查 Azure Functions 的同步排程狀態。
```
