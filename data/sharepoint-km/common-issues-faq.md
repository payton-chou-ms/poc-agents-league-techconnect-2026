# 常見問題與解法 (FAQ)

> 最後更新: 2026-01-20 | 維護者: IT 支援團隊

---

## Q1: 庫存數字與實際不符怎麼辦？

**問題描述**: 系統顯示有庫存，但門市回報實際無貨。

**可能原因**:
1. 供應商資料同步延遲（API timeout）
2. 門市盤點尚未回傳
3. 退貨未即時入庫

**解決步驟**:
1. 確認 Fabric Lakehouse 資料最後更新時間
2. 檢查供應商 API 同步狀態 (`GET /api/v2/supplier/sync-status`)
3. 若 API timeout > 30 秒，參考「供應商資料同步說明」排查
4. 通知門市進行緊急盤點

---

## Q2: 跨區域庫存調度流程是什麼？

**流程**:
1. 確認目標區域缺貨品項與數量
2. 查詢其他區域可調撥庫存
3. 提交調撥申請（需主管審核）
4. 物流安排出貨，預估到達時間
5. 更新雙邊庫存記錄

**注意**: 跨國調度需額外計算關稅與海運/空運成本。

---

## Q3: 供應商同步 API 回傳 500 錯誤怎麼辦？

**緊急處理**:
1. 查看 Azure Monitor 的 API 日誌
2. 確認供應商端服務是否正常
3. 若為我方問題，重啟同步服務：`systemctl restart supplier-sync`
4. 確認同步恢復後手動觸發全量同步

---

## Q4: 產品大量缺貨如何啟動應急流程？

**應急流程**:
1. 📊 確認缺貨範圍（單一區域 or 全球）
2. 📚 查詢知識庫是否有類似歷史案例
3. 🔧 檢查是否有系統 Bug 導致資料異常
4. 🚚 啟動緊急補貨物流
5. 📝 產生事件報告
6. 📅 召開跨部門會議
